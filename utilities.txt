The app is not using any traditional algotithm. what we are using:

- Cache-First-Pattern: This is a decision-making pattern our content service follows every single time
                        a student requests content.

                        is the content already in the cache?
                        if yes -> return it immediately
                        else: generate it & save it -> return it


- Prompt Engineering: When we build AI Prompt dynamically from database data, we are following a structured process:
                        Fetch subtopic data from database
                            Extract title, topic, keywords
                                fill into prompt template
                                    send to Claude API


- The Intelligence Layer:(Claude) -> Claude itself is powered by a Large Language Model which is built on top of very complex
                                     algorithms, specifically a type called a Transformer neural network. But that is Claude's
                                     internal world. We are just the caller, we send a prompt and receive a response
